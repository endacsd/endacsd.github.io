  • 存储速度

  • 运算速度

微处理器的发展

微处理器使用一片或少数几片大规模集成电路组成的中央处理器

1 四位微处理器

1970年代初

典型产品 Intel 4004, Inteal4040

字长 4位

特点 指令系统比较简单，运算功能较差，价格低廉

应利 面向家电，计算器和二次仪表

2 八位微处理器

1974-1977

字长 8位

特点 指令系统比较完善，运算速度提高一个数量级，寻址能力有所增强，有中断和DMA,汇编，BASIC,FORTRAN

应用 面向家电，智能仪表，工业控制

3 十六位微处理器

1978-1980

特点 指令系统丰富，采用多级中断，多种寻址方式，段式存储结构，配有功能强大的系统软件

应用 工业控制

8086微处理器的构成

内部结构

由两个独立的逻辑单元组成:

  1. 总线接口单元BIU: Bus Interface Unit,完成所有总线操作。
  2. 执行单元EU: Execution Unit, 执行指令。

二者并行工作

8086/8088

EU相同，BIU有差别

  1. 8086: 外部DB 16 位， 指令队列为6个字节
  2. 8088: 外部DB 8 位， 指令队列为4个字节

BIU

BIU 组成

  1. 4个16位段寄存器
  2. 16位指令指针
  3. 6字节指令队列
  4. 总线控制逻辑
  5. 20位地址加法器

BIU功能

  1. 取指令
  2. 形成物理地址
  3. 传送数据，实现CPU与内存,I/O端口间的数据传送

指令预取策略:

当指令队列出现2个以上空字节时(8086)，自动按CS:IP 组成20位物理地址，到存储器去取指令，一次取2个字节

EU

EU 组成:

  1. 16位ALU
  2. 16位标志寄存器
  3. EU控制电路
  4. 8个16位通用寄存器

EU 功能: 完成指令所规定的操作

  1. 从BIU指令队列读取指令
  2. 由EU控制电路译码分析
  3. 在EU种计算操作数的16位偏移地址送BIU,由BIU的$\sum$形成20位物理地址
  4. 将取来的操作数送ALU进行指令运算
  5. 结果送寄存器或送到BIU放到内存
  6. 本次操作状态放到标志寄存器中

结构特点

  1. 由BIU,EU 两个独立单元组成
  2. 取指和执行重叠

8086的两种组成模式

8086总线周期

工作时序: 微处理器按照一定的时序工作

时钟周期: CPU主频每个时钟脉冲的持续时间。用一个T表示

总线周期: CPU通过总线进行一次读或写的过程

指令周期: 执行一条指令所需要的时间

总线请求相应:利用总线传输数据时，总线控制部件必须获得总线的控制权，总线请求时总线控制部件发出的占用总线的请求信号，总线相应时但钱控制总线的部件或总线控制器在接收到总线请求后，给出的应答信号

中断相应周期: 中断响应十CPU接受中断请求后的处理过程，在响应中断时，CPU在当前指令结束后，插入两个总线周期，发出中断答应，并通过总线获取中断类型码

8086 时钟状态

8086一个基本的总线周期由4个时钟周期组成。每个时钟称为T状态，用T1，T2,T3,和T4表示

T1状态: CPU发出地址。CPU存储器地址或I/O端口的地址送上地址总线

T2状态:扯地址，发控制信号，进行读写准备，CPU撤销地址/数据，地址/状态复用线上的地址，地址/数据复用线浮置，地址/状态复用线输出状态，即复用信号在这个周期切换，读写控制信号有效。

T3状态: 地址/数据线上出现数据，写操作，CPU提供数据;读操作，等待存储器或I/O提供数据，检查READY信号，未就绪，插入TW,TW操作与T3相同

T4状态:完成数据读/写，控制信号无效，结束总线操作

输入输出的寻址方式

  1. 存储器映像方式

    把一个I/O端口看作一个存储单元，相当于给每一个I/O端口分配一个存储器地址

    • 优点
      • 指令丰富
      • I/O端口空间大
      • 寻址的控制逻辑地址比较简单
    • 缺点
      • I/O端口占用一部分存储器地址空间，使可用的内存空间相对减少
      • 对I/O端口的访问和对存储器的访问一样，必须对全部地址译码，译码电路会比较复杂
      • 存储器操作指令的机器码比较长，需要较长的执行时间
      • 不易区分

  2. I/O映像方式

    在这种寻址方式中，I/O端口空间与存储器各自独立，互不干涉，互不影响

    在指令操作上，对存储单元的一般性传送使用MOV指令，而对I/O端口的传送操作，使用系统专门提供的一组I/O指令，即IN和OUT指令

    • 优点
      • I/O空间与存储器各自独立
      • 单独指令，更清晰
      • 地址线少，译码电路简单
      • 指令格式段，执行时间快
    • 缺点
      • 专门的I/O指令，设计灵活性差
      • 端口数目受到限制
      • 使用专门的控制线，限制了引脚，也增加了控制逻辑的复杂性

程序中断方式

为了使得CPU和外设已经外设之间能并行工作

主程序中某一时刻安排启动某一台外设的指令，然后CPU继续执行其主程序，当外设完成数据传送的准备后，向CPU发出”中断请求”信号，在CPU可以相应中断的条件下，中断(即为暂停)线性主程序的执行，而转去执行中断服务程序，在中断服务程序中完成一次CPU与外设之间的数据传送，传送完成后仍返回被中断的断点处继续执行主程序

中断方式传送效率不高，CPU执行程序来实现数据传送，CPU要保护断点，转入中断服务程序，每次传送一个字节(或一个字)

DMA方式: 在外设和内存之间直接传送数据，即直接存储器传输方式

基本方法

  1. CPU停机方式

    进行DMA传送时，DMA控制器向CPU发出总线控制信号,迫使CPU在现行的总线周期结束后，示器地址总线，数据总线和部分控制总线处于高阻态，从而让出对总线的控制权，并给出DMA相应信号，DMA控制器接到高相应信号后，就可以对总线进行数据传送的控制工作，知道DMA操作完成，CPU再恢复对总线的控制权，继续执行被中断的程序

  1. 周期扩展

    进行DMA操作时，由DMA控制器发出请求信号给时钟电路，时钟电路把供给CPU的时钟周期加宽，而提供给存储器和DMA控制器的时钟周期不变，用这种方法进行DMA操作，一次只能传送一个字节

  2. 周期挪用

    利用CPU不访问内存的那些走起来实现DMA操作，此时DMA操作使用总线不用通知CPU也不会影响CPU的工作

DMA控制器的功能

  1. 当外设准备就绪，希望进行DMA操作时，会像DMA控制器发出DMA请求信号，DMA控制器接到此信号后，应能向==CPU发出总线请求信号==
  2. CPU接到总线信号后，如果允许，则会发出DMA相应信号，从而CPU放弃对总线的控制，这时DMA控制器应能实现对总线的控制
  3. DMA控制器得到总线控制权以后，要往地址总线发送地址，修改所用的存储器的地址指针
  4. 在DMA传送期间，DMA控制器应能发送存储器或接口的读/写控制信号
  5. 能统计传送的字节数，并且判断DMA传送是否结束
  6. 能够向CPU发出DMA结束信号，将总线控制权交还给CPU

主从模态

DMA 控制器可以作为I/O端接收CPU的读写操作，也可以代替CPU占有总线，控制外设与存储器之间传送数据，它充分体现了DMA控制两大特性，即总线的主控性和总线的从属性，按这两大特性，它也就由两种工作模态，主态方式和丛态方式

主态方式

DMA控制器是总线的控制这，此时8237A是主模块，它如同CPU一样，掌握总线的控制权，可对涉及的外设端口或存储器单元进行读写操作

从态方式

PCU是总线的控制者，DMA控制器不过是普通的一个外部设备，有若干个端口而已，它的低位同一般的I/O芯片是一样的，所以，此时8237A是从模块

传送方式
  • 单字节传送方式
  • 数据块传送方式
  • 请求传送方式
  • 级联传送方式

1 单字节传送方式

一次只传送一个字节，然后释放总线，若又有外设DMA请求，8237A再向CPU发出下一次总线请求HRQ,获得总线控制权后，再传送道下一个字节数据

注意

再DACK有效前，DREQ应该有效

即使DREQ在传送一个字节后DMA控制器释放总线，但由于总线相应后HRQ也将变为无效，并在传送一个字节后DMA控制器释放总线，但由于DREQ一直有效，HRQ很快百再次变成有效，在芯片接收道新得HLDA后，下一个字节又开始传输，显然，在两次DMA传送之间至少执行一个完整的机器周期，在此期间，完成可能相应另一个高优先级的DMA请求

每次传送后，当前字节计数器减1，当前地址寄存器减1或则加1，当当前字节计数器减1由0变为FFFFH时，发出有效信号(产生终止计数TC信号)，如果通道变成设为自动初始化方式，则自动地重新装入计数值和地址寄存器

2数据块传送方式

数据块传送方式，相应一次DMA请求，将完成设定的字节数的全部传送，当字节计数器减1由0变为FFFFH时，产生TC有效信号，使8237A将总线控制权交还给COU从而结束DMA操作方式，外部有效的信号也可以总结DMA传送

在DACK变为有效之前，DREQ信号必须保持有效，一旦DACK有效，不管DREQ如何，8237A一直不放弃总线控制权，几百年是在传送过程种，DREQ变为无效，8237A也不会释放总线，只是暂停数据的传送，知道通道请求信号再次有效后，8237A又继续进行数据传送，直到真个数据块全部传送晚，才会退出DMA操作，将总线控制权交还CPU

PC机不饿能采用这种方式，否则会影响道动态存储刷新和磁盘驱动器的数据传送，他们都不允许另一个DMA传送长期占用总线

3请求传送方式

请求传送方式 又叫做查询方式，类似数据块传送，但每传送一个字节后，检测DREQ状态，若无效则停止，若有效则继续DMA传送，

1） DREQ 变为无效

2） 字节计数器减1由0变为FFFFH，产生TC信号

3）外界输入有效信号

当DREQ无效时，8237A停止传送，内部的当前寄存器和当前字节计数器还保留当时的数值，一旦外设准备好要传送的新的数据，可以再次使得DREQ变为有效，就可以传送继续下去，当DREQ无效时，8237A停止传送，此时释放总线，当DREQ重新有效时，将重新开始一次DMA请求过程

4 级联方式

这种方式允许连接一个以上的芯片来扩展DMA通道的格式

连接方法 将扩展的DMA芯片的HRQ和HLDA分别连到主片的某个通道的DREQ和DACK

作用

优先级管理的作用，即从片的4个DMA通道纳入主片的优先级管理机制

向CPU输出HRQ和HLDA

传送类型

三种

1） DMA 读

8237A输入有效的MEMR和IOW信号，把存储器的数据读到I/O设备

2) DMA写8237A输出有效的IOR和MEMW信号，把I/O设备的数据写到存储器

3） DAM校验

这是一种伪传输，实际上是校验作用 对存储器和I/O设备的控制线均处于无效状态

编码优先级

固定优先级，循环优先级

DREQ0最高

循环优先级的4个通道的优先级顺序是可变的，当某个通道申请DMA请求并被服务相应后，它就被指定为最低优先级，它的下一级成为了最高优先级

禁止嵌套

自动化初始方式

通过对方式字寄存的编程，可设置某个通道为自动初始化方式

自动初始化方式

当该通道完成一个数据传送并且产生信号时(可能是由内部TC产生，也可能是外部产生)

用基地址寄存器和基字节数计数器的内容，使相应的当前地址存储器和当前字节数计数器恢复初值

当前地址寄存器和当前字节数计数器的最初值，是由CPU在初始化编程时写入的（这个值最初也同时写入到了基地址寄存器和基字节计数器），但在DMA传送过程中，而基地址寄存器和当前字节数计数器的内容不被修改，而基地址寄存器和基字节计数器的内容维持不变(除非从新编程) 在自动初始化以后，通道就作好了进行另外一次DMA传送的准备

存储器到存储器的传送

利用这种方式，可以使得数据块从一个存储空间传送到另外一个存储空间，将程序的影响和传输时间减到最小

传送方向 0地址寄存器 -> 1地址寄存器

计数方法 采用#1字节计数器

启动方法 设置一个#0的软件DREQ启动

Pentium的中断机制

最多可以处理256个中断源，每个中断源都有一个8位二进制数的中断类型码

中断源分为4中：

  • 可屏蔽中断(INTR)
  • 非屏蔽中断(NMI)
  • 软件中断(INTO,INT3,INT n )
  • 异常
中断向量表

图来

  • 中断向量： 中断服务程序的入口地址
  • 中断向量表: 存放中断服务程序入口地址的表格
  • 中断向量表位于存储器地址空间的低地址端，1K字节，地址为00000H-003FFH,每4个字节存放一个中断服务程序的入口地址
  • 较高地址两个字节存放中断服务程序入口地址的段内偏移量，这4个单元的最低地址称为向量地址，其值为对应的中断类型码乘4
  • 在实模式下，该表称为中断向量表，在保护模式下，该表称为中断描述符表
可屏蔽中断INTR

INTR引脚: 高电平有效，并且当前的中断允许标志位IF为1时，CPU在当前指令执行结束后，会相应INTR引脚的中断请求

Pentium要求可屏蔽中断请求信号时一个高电平信号，并且，INTR信号的高电平必须维持到CPU相应中断后才结束

非屏蔽中断NMI

非屏蔽中断请求信号NMI用于重要事件处理，入电源掉电，存储器读写出错，总线奇偶位出错等

NMI中断: 不受中断允许标志IF的影响。上升沿触发。中断类型固定为2

软件中断

软件中断包括INTO(溢出)，INT3(断点),INT n, BOUND(界限比较) 指令

软件中断的特点

  • 认为设置
  • 中断类型码包含在指令中，所以不执行中断相应总线周期
  • 除BOUND指令外，中断返回地址指向软件中断指令下一条指令
异常

既不是外部硬件产生的，也不是用软件指令产生的，而是在CPU执行一条指令过程中出现的错误或检测到的异常情况自动产生的

除了单步(陷阱)中断外，异常中断返回地址指向程序中发生中断的地方，而不是指向产生中断指令的下一条指令

  • 类型0除法出错
  • 类型1单步(陷阱)
  • 类型6无效操作码
