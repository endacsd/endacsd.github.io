微处理器的发展

微处理器使用一片或少数几片大规模集成电路组成的中央处理器

1 四位微处理器

1970年代初

典型产品 Intel 4004, Inteal4040

字长 4位

特点 指令系统比较简单，运算功能较差，价格低廉

应利 面向家电，计算器和二次仪表

2 八位微处理器

1974-1977

字长 8位

特点 指令系统比较完善，运算速度提高一个数量级，寻址能力有所增强，有中断和DMA,汇编，BASIC,FORTRAN

应用 面向家电，智能仪表，工业控制

3 十六位微处理器

1978-1980

特点 指令系统丰富，采用多级中断，多种寻址方式，段式存储结构，配有功能强大的系统软件

应用 工业控制

4 三十二位微处理器

1980年带初开始，之后十余年迅速发展

5 六十四位处理器

8086微处理器的构成

内部结构

由两个独立的逻辑单元组成:

  1. 总线接口单元BIU: Bus Interface Unit,完成所有总线操作。
  2. 执行单元EU: Execution Unit, 执行指令。

二者并行工作

8086/8088

EU相同，BIU有差别

  1. 8086: 外部DB 16 位， 指令队列为6个字节
  2. 8088: 外部DB 8 位， 指令队列为4个字节

BIU

BIU 组成

  1. 4个16位段寄存器
  2. 16位指令指针
  3. 6字节指令队列
  4. 总线控制逻辑
  5. 20位地址加法器

BIU功能

  1. 取指令
  2. 形成物理地址
  3. 传送数据，实现CPU与内存,I/O端口间的数据传送

指令预取策略:

当指令队列出现2个以上空字节时(8086)，自动按CS:IP 组成20位物理地址，到存储器去取指令，一次取2个字节

EU

EU 组成:

  1. 16位ALU
  2. 16位标志寄存器
  3. EU控制电路
  4. 8个16位通用寄存器

EU 功能: 完成指令所规定的操作

  1. 从BIU指令队列读取指令
  2. 由EU控制电路译码分析
  3. 在EU种计算操作数的16位偏移地址送BIU,由BIU的$\sum$形成20位物理地址
  4. 将取来的操作数送ALU进行指令运算
  5. 结果送寄存器或送到BIU放到内存
  6. 本次操作状态放到标志寄存器中

结构特点

  1. 由BIU,EU 两个独立单元组成
  2. 取指和执行重叠

8086的两种组成模式

8086总线周期

工作时序: 微处理器按照一定的时序工作

时钟周期: CPU主频每个时钟脉冲的持续时间。用一个T表示

总线周期: CPU通过总线进行一次读或写的过程

指令周期: 执行一条指令所需要的时间

总线请求相应:利用总线传输数据时，总线控制部件必须获得总线的控制权，总线请求时总线控制部件发出的占用总线的请求信号，总线相应时但钱控制总线的部件或总线控制器在接收到总线请求后，给出的应答信号

中断相应周期: 中断响应十CPU接受中断请求后的处理过程，在响应中断时，CPU在当前指令结束后，插入两个总线周期，发出中断答应，并通过总线获取中断类型码

8086 时钟状态

8086一个基本的总线周期由4个时钟周期组成。每个时钟称为T状态，用T1，T2,T3,和T4表示

T1状态: CPU发出地址。CPU存储器地址或I/O端口的地址送上地址总线

T2状态:扯地址，发控制信号，进行读写准备，CPU撤销地址/数据，地址/状态复用线上的地址，地址/数据复用线浮置，地址/状态复用线输出状态，即复用信号在这个周期切换，读写控制信号有效。

T3状态: 地址/数据线上出现数据，写操作，CPU提供数据;读操作，等待存储器或I/O提供数据，检查READY信号，未就绪，插入TW,TW操作与T3相同

T4状态:完成数据读/写，控制信号无效，结束总线操作

指令前缀: 说明指令的相关特性

地址大小地址: 改变默认的地址长度

接口与端口

接口： 在微型计算机系统中，CPU与外部设备之间的联系，需要有特定的硬件连接和相应的控制软件。

端口: 接口电路中那些信息传送，可有程序寻址并进行读写操作的寄存器

若干个端口加上相应的逻辑才构成接口

接口的功能

  1. 地址译码或设备选择

    接口必须进行地址译码，从而产生设备选择信号，以使微处理器和指定的外部设备交换信息

  2. 数据缓冲和锁存

    在微计算机系统总，数据总线时系统各部分之间公用的双向总线，所有设备分时复用，所以，无论是存储器还是外部设备，都不能长期占用数据总线，只允许被选中的设备在读/写周期内可用其他传送数据

  3. 信息格式与电平转换

    接口应该具有信号传送格式，信号类型，信号电平的转能力

  4. 数据传送的协调

    CPU工作是有一定时序的，CPU与外部设备交换数据必须采用一定传送方式进行控制

输入输出的寻址方式

  1. 存储器映像方式

    把一个I/O端口看作一个存储单元，相当于给每一个I/O端口分配一个存储器地址

    • 优点
      • 指令丰富
      • I/O端口空间大
      • 寻址的空寂逻辑地址比较简单
    • 缺点
      • I/O端口占用一部分存储器地址空间，使可用的内存空间相对减少
      • 对I/O端口的访问和对存储器的访问一样，必须对全部地址译码，译码电路会比较复杂
      • 存储器操作指令的机器码比较长，需要较长的执行时间
      • 不易区分

  2. I/O映像方式

    在这种寻址方式中，I/O端口空间与存储器各自独立，互不干涉，互不影响

    在指令操作上，对存储单元的一般性传送使用MOV指令，而对I/O端口的传送操作，使用系统专门提供的一组I/O指令，即IN和OUT指令

    • 优点
      • I/O空间与存储器各自独立
      • 单独指令，更清晰
      • 地址线少，译码电路简单
      • 指令格式段，执行时间快
    • 缺点
      • 专门的I/O指令，设计灵活性差
      • 端口数目受到限制
      • 使用专门的控制线，限制了引脚，也增加了控制逻辑的复杂性
使用门电路组合法进行端口的地址选择

采用与门，或门，非门等作为基本的组合元件

端口都有一个芯片选择信号，简称片选信号，多数为低电平有效

端口选择的目的，是当地址线出现某种信息组合时，在端口地址选择电路的输出端会产生一个有效信号，该信号连到器件的控制端，使器件参数动作，从而完成读/写操作

有效信号的四种状态: 高电平，低电平，上跳沿，下降沿

输入输出控制方式

四种基本控制方式

  • 程序查询方式

    CPU通过查询I/O设备的状态，断定哪个设备需要服务，然后转入相应的服务程序

  • 程序中断方式

    当I/O设备需要CPU为其服务时，可以方式中断请求信号INTR,CPU接到请求信号后，中断正在执行的程序转去为该设备服务，服务完毕，返回原来被中断的程序继续执行

  • 直接存储器取(DMA)方式

    采用这种方式时，在DMA控制器的管理下,I/O设备和存储器直接交换信息，而不需要CPU的介入

  • I/O处理机方式

    引入I/O处理机，全部的I/O操作由I/O处理机独立完成

程序查询式

分为无条件传送和查询传送方式

查询: 就是询问外部设备的工作状态，通过这一状态来判断外设是否具备了与CPU交换数据的能力

对于输入而言，这个状态hi是输入的数据是否已经就绪

对于输出而言，遮盖状态指示输出设备的数据接收寄存器是否已空

缺点 CPU和外设只能串行工作，各外设之间也只能串行工作

程序中断方式

为了使得CPU和外设已经外设之间能并行工作

主程序中某一时刻安排启动某一台外设的指令，然后CPU继续执行其主程序，当外设完成数据传送的准备后，向CPU发出”中断请求”信号，在CPU可以相应中断的条件下，中断(即为暂停)线性主程序的执行，而转去执行中断服务程序，在中断服务程序中完成一次CPU与外设之间的数据传送，传送完成后仍返回被中断的断点处继续执行主程序

DMA方式

中断方式传送效率不高，CPU执行程序来实现数据传送，CPU要保护断点，转入中断服务程序，每次传送一个字节(或一个字)

DMA方式: 在外设和内存之间直接传送数据，即直接存储器传输方式
