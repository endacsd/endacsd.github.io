操作系统概述

操作系统概念

操作系统地位

硬件抽象层(HAL) 之上，所有其他软件层之下

系统库可调用操作系统，执行硬件指令

应用程序可以调用lib和操作系统，执行硬件指令

操作系统的作用

管理系统中软件硬件资源

CPU 一个CPU,多个可运行的程序

内存

设备 分配，驱动

文件 实现文件系统，支持文件操作

为用户提供良好的服务

API

GUI,行式命令(SHELL)

JCL(job Control Language)

操作系统定义

操作心痛式位于硬件层(HAL)之上，所有其他软件层之下的一个系统软件，式管理系统中各种软硬件资源，方便用户使用计算机系统的程序集合

操作系统的历史

操作系统的产生

手工操作阶段（20世纪40年代，无操作系统）

  • (程序，数据)$\rightarrow$ 穿孔机$\rightarrow$纸带；
  • 穿孔自带$\rightarrow$光电机$\rightarrow$机器内存；
  • 控制台开关启动第一条指令

（调试程序需要修改自带，或动态修改）

（输入也需要安装纸带）

  • 运行结构在电传打字机上输出

缺点

资源独占

手工费时

批处理阶段（20世纪50年代，操作系统雏形）

(1) 联机批处理

使用监督程序，直接处理多个作业

摆脱了人工干预

缺点 I/O操作慢，主机等待时间长

(2) 脱机批处理

用卫星机来处理I/O

优点 减少了主机等待I/O操作时间

缺点 人工搬动磁带 额外的卫星机

执行系统阶段(20世纪60年代初期)

通道： 专门用于控制I/O设备的处理机，即I/O处理机

  • 有自己的指令系统和运控部件
  • 与主吉共享内存
  • 接受主CPU委托执行通道程序，完成I/O;
  • 通道I/O操作与主CPU并行
  • 通道完成I/O时，向主机发中断请求

非联机，非脱机

假脱机

操作系统的完善

多到批处理系统(60年代初期)

  • 执行系统： 单道作业，资源利用不充分；
  • 多道批处理： 主机中同时放多个作业，最大限度提高资源利用率

单道到多道 不是量的变换，是质的飞跃

带来的问题： 互斥，同步，通讯，死锁，饥饿，饿死

多道批处理出现，标志操作系统走向成熟

分时系统（60年代初，中期）

程序员提出 联机操作的要求

一台主机，多个终端，交互式工作

实时系统(60年代中期)

背景： 计算机应用领域扩大

要求 ： 满足时间约束条件

通用操作系统(60年代后期)

上述三类系统的结合

计算机系统的发展

网络操作系统和分布式操作系统

单用户多任务的操作系统

多处理器的并行操作系统

微内核操作系统体系结构，产生了嵌入式和智能卡操作系统

多核技术 多核的并发控制，多核下的进程调度

操作系统特性

并发性

多个程序在宏观上同时向前推进

并发(concurrent) vs. 并行(parallel)

  • 用户程序与用户程序并发
  • 用户程序与OS并发
  • OS与OS并发

共享性

多个程序公用系统中的各种软硬件资源

在操作系统的协调和控制下

异步性(随机性)

多个程序以不可预知的速度向前推进

虚拟性

把一个实的CPU改为多个虚的CPU

内存+外存$\rightarrow$ 虚存

独占设备+共享设备$\rightarrow$ 虚拟设备(数量多，速度快)

操作系统运行环境

进程、线程与作业

多通道程序设计

吞吐量是衡量系统效率的一个尺度，定义为

单通道的缺点

资源利用率极低，主要体现在

设备资源利用率低

内存资源利用率低

处理器资源利用率低

多道程序提出的设计

设备资源利用率提高

内存资源利用率提高

处理器资源利用提高

自然表达一个程序内在的可并行性

多道程序设计的问题

处理器资源管理问题

可运行程序和处理器资源竞争问题，需要对资源进行管理，实现处理器资源在各个程序之间的调度

内存资源管理问题

程序运行过程种内存地址的变化，程序使用相对地址，操作系统需要负责存储空间的分配和管理，实现程序的重定位，此外，还要防止内存种多道程序之间的互相干扰或对操作系统的干扰，要实现对存储空间的保护

设备资源管理问题

要求操作系统确定适当的分配策略，并据此对资源加以管理

进程的引入

在多道程序系统种运行的程序时处于时断时续的状态之中，一个程序获得处理器资源后向前推进，当他需要某种资源而未得到时暂停，以后得到资源再继续向前推进

程序暂停时，需要将其现场信息作为断点保存起来，以便于再次推进时能够恢复上次暂停时的现场信息的区域，这个区域并不是程序的组成部分，因此就需要一个能够更准确描述多道程序系统中执行程序的术语，这就是进程

进程的概念

进程是程序的一次执行

进程是可以参与并发执行的程序

进程是程序与数据一道通过处理器执行时所发生的活动

所谓进程，就是一个程序再给定的空间和初始环境下，在一个处理器执行的过程

定义2-1 进程时育有一定独立功能的程序关于一个数据集合的一次运行活动

进程状态及状态状态

进程的状态

进程再其生存周期内可能处于以下3中基本状态之一

  • 运行(run) 态： 进程占有处理器资源，正在运行，显然，再单处理器系统中任一时刻只能有一个进程处于此种状态

  • 就绪(ready)态： 进程本身具备运行条件，但是由于处理器的数量少于可运行进程的数量，暂未投入运行，即相当于等待处理器资源

  • 等待(wait)态：也称为挂起(suspend)态、阻塞(block)态、睡眠(sleep)态.进程本身不具备运行条件，即使分给其处理器资源也不能运行。进程正在等待某一事件得发生，如等待某一资源被释放，等待该进程相关得数据传输得完成信号等。

运行，就绪，等待是进程最基本得3种状态，对一个具体得系统来说，为了实现某种设计目标，进程状态得数量可能多于3个

状态转换

进程控制块

多道程序系统中运行的程序需要有一个断点现场保存区域，这个区域就设在进程控制块(process control block,PCB) 中

定义 进程控制块是标志进程存在的数据结构，其中包含系统对进程进行管理所需要的全部信息

一般包括了

  • 进程标识
  • 用户标识
  • 进程状态
  • 调度参数
  • 现场信息
  • 家族联系
  • 程序地址
  • 当前打开文件
  • 消息队列文件
  • 资源使用情况
  • 进程队列指针

进程的组成与上下文

进程由进程控制块和程序组成。

进程控制块，透明化

程序

数据一般包括静态变量，动态堆和动态栈。堆用来保存动态变量，栈用来保存动态变量，栈用来保存用户子程序互相调用时的参数，局部变量，返回值，断点等。
